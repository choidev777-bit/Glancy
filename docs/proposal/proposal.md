## 1. 서비스 개요

### 1-1. 제품명

**Glancy** — Skills.md 기반 범용 투자 분석 대시보드

Glancy는 "한눈에 본다"는 의미에서 지은 이름이다. 흩어진 투자 데이터, 지표, 차트, 인사이트를 한 화면에서 빠르게 파악하게 만든다는 제품 방향을 담고 있다.

### 1-2. 핵심 가치 제안

**한 줄 요약:** Skills.md 문서가 곧 시스템이 되는 투자 분석 도구

Glancy는 한국주식, 미국주식, ETF, 암호화폐, 글로벌지수, 사용자 업로드 데이터를 하나의 분석 체계로 처리하는 범용 투자 데이터 대시보드다. 사용자는 사전 연결된 카테고리 탭을 통해 즉시 시장 데이터를 분석할 수 있고, CSV/JSON 업로드를 통해 본인의 데이터도 같은 방식으로 분석할 수 있다.

본 서비스는 다음 두 가지 시나리오를 모두 지원한다.

1. **카테고리 탭으로 즉시 탐색** — 한국주식 / 미국주식 / ETF / 암호화폐 / 글로벌지수 / 업로드 탭을 통해 별도 설정 없이 분석 시작
2. **사용자 데이터 자동 분석** — CSV/JSON 업로드 시 데이터 유형을 자동 감지하고, 유형에 맞는 분석과 시각화를 자동 생성

### 1-3. 차별점 — Skills.md를 런타임 설정 파일로 활용

일반적인 접근은 Skills.md를 "개발 시 AI에게 준 가이드 문서"로 사용하는 것이다. Glancy는 한 단계 더 나아가 Skills.md를 **런타임 설정 파일**처럼 사용한다.

예를 들어 사용자가 `indicators.md`의 RSI 기준을 바꾸면 게이지와 인사이트가 새 기준으로 다시 계산되고, `theme.md`의 색상 토큰을 바꾸면 대시보드와 차트 스타일이 즉시 바뀐다. 이는 "문서가 시스템을 구동한다"는 것을 심사자가 직접 체험할 수 있는 구조다.

### 1-4. 타겟 사용자

- 개인 투자자: 여러 자산군을 한 화면에서 비교하고 기술적/기본적 상태를 빠르게 판단
- 데이터 분석가: CSV/JSON 데이터셋을 업로드해 자동 시각화와 통계 분석 확인
- 투자 입문자: RSI, MACD, 이동평균, 밸류에이션 같은 숫자를 자연어 인사이트로 이해
- 심사자/참가자: 별도 API 키 없이 URL 접속만으로 대시보드와 Skills.md 기반 자동화를 확인

## 2. 분석 흐름 설계

### 2-1. 전체 데이터 파이프라인

```text
[입력]                    [Skills.md 모듈]              [출력]
────────────────────────────────────────────────────────────────────
카테고리 탭 선택      ->   data.md           ->   표준화된 MarketData
또는 CSV/JSON 업로드       데이터 소스 레지스트리

                      ->   indicators.md     ->   계산된 지표 + 시그널
                           지표 계산 규칙

                      ->   insights.md       ->   자연어 인사이트 텍스트
                           규칙 기반 해석

                      ->   charts.md         ->   ChartSpec + VisualizationBundle
                           데이터 유형별 시각화 선택

                      ->   layout.md         ->   페이지 구성
                           반응형 / 상태 처리

                      ->   theme.md          ->   디자인 토큰 적용
                           다크/라이트/차트 색상

                                             ->   최종 대시보드 렌더링
```

### 2-2. Orchestration — main.md

`main.md`는 시스템 진입점이다. 사용자 입력이 사전 연결된 카테고리 탭인지, CSV/JSON 업로드인지에 따라 어떤 모듈을 어떤 순서로 호출할지 정의한다. 또한 데이터 실패, 지표 계산 실패, 데이터 유형 감지 실패 시 fallback 정책을 제공한다.

### 2-3. 분석 유형 분기

| 입력 유형 | 분석 종류 | 기본 시각화 |
|----------|----------|-------------|
| 한국/미국 주식 | 기술적 분석 + 기본적 분석 | 캔들, MA, RSI, MACD, 게이지, 펀더멘털 카드 |
| ETF / 암호화폐 / 지수 | 기술적 분석 | 캔들, MA, RSI, MACD, 피벗, 게이지 |
| OHLCV CSV | 기술적 분석 | 캔들, 거래량, 지표 테이블 |
| 포트폴리오 CSV | 비중/집중도 분석 | 도넛 차트, 집중도 막대 |
| 다종목 비교 CSV | 상대 성과/상관관계 분석 | 정규화 수익률 라인, 상관관계 히트맵 |
| 수익률 시계열 CSV | 성과/위험 분석 | 누적수익률, MDD, 월별 수익률 히트맵 |
| 가격 시계열 CSV | 가격 흐름/낙폭 분석 | area/line, drawdown |

## 3. 대시보드 구성

### 3-1. UI 계층 구조

```text
[헤더] 로고 + 데모 안내 + 다크/라이트 토글
──────────────────────────────────────────────────────────
[카테고리 탭] 한국주식 | 미국주식 | ETF | 암호화폐 | 글로벌지수 | 업로드
──────────────────────────────────────────────────────────
[종목 정보] 종목명 / 현재가 / 등락률 / 거래량 / 시가총액 / 52주 고저
──────────────────────────────────────────────────────────
[분석 탭] 요약 | 기술적 분석 | 기본적 분석
──────────────────────────────────────────────────────────
[요약]        종합 게이지 + 인사이트 + 핵심 카드
[기술적 분석] 게이지 3개 + 캔들차트 + RSI/MACD + 지표/피벗 테이블
[기본적 분석] 밸류에이션 / 수익성 / 성장성 / 재무건전성 / 주주환원
[업로드]      자동 감지 + 데이터 유형별 자동 시각화 대시보드
```

### 3-2. 와우 모먼트

1. **카테고리 즉시 전환** — 한국주식, 미국주식, ETF, 암호화폐, 글로벌지수를 같은 UI에서 탐색
2. **실시간/준실시간 데이터** — 암호화폐는 Binance WebSocket으로 실시간에 가깝게 갱신하고, 주식/ETF/지수는 카테고리 전환 시 최신 데이터를 조회하며 캐시/fallback으로 안정성을 확보
3. **Skills.md 기반 자동 해석** — 외부 LLM API 없이 규칙 기반 자연어 인사이트 생성
4. **CSV 업로드 자동 분석** — 데이터 유형 감지 후 동일 분석 시스템으로 처리
5. **Visualization Intelligence** — 업로드 데이터 유형에 따라 도넛, 히트맵, drawdown, 정규화 비교 차트 자동 선택
6. **Skills Runtime Demo** — `theme.md`, `indicators.md` 수정 시 대시보드가 즉시 변화

### 3-3. 사용자 파라미터 변경

이동평균 기간/크로스 기준, RSI, MACD, Stochastic, Bollinger Bands, Williams %R, CCI, ATR, ROC, OBV의 주요 기간과 임계값을 사용자가 즉석에서 변경할 수 있다. 변경값은 `indicators.md` 런타임 오버라이드로 처리되어 게이지, 지표 테이블, 차트 보조선, 인사이트가 다시 계산된다.

### 3-4. 데이터 소스

| 카테고리 | 데이터 소스 | API 키 |
|---------|-------------|--------|
| 한국 주식 | pykrx + DART API | 서버 환경변수로 관리, 심사자는 별도 키 불필요 |
| 미국 주식 | yfinance | 불필요 |
| ETF | yfinance ETF | 불필요 |
| 암호화폐 | Binance + CoinGecko | 불필요 |
| 글로벌 지수 | yfinance (^KS11, ^GSPC 등) | 불필요 |
| 사용자 업로드 | CSV/JSON | 불필요 |

외부 API가 실패할 경우 sample fallback 데이터로 자동 전환하여 심사자가 기능을 계속 확인할 수 있도록 한다.

## 4. Skills.md 설계 방향

### 4-1. 모듈화 구조

운영진 규정상 여러 개의 Skills.md 제출이 가능하고 기능별 구분이 권장되므로, Glancy는 다음 7개 모듈로 나눈다.

```text
skills/
├── main.md          # Orchestrator: 실행 흐름, 모듈 간 호출 정의
├── data.md          # Data Layer: 6개 데이터 소스 통합, 표준화
├── indicators.md    # Computation Layer: 기술/기본 지표 계산 규칙
├── insights.md      # Narrative Layer: 규칙 기반 자연어 해석
├── charts.md        # Visualization Layer: 데이터 유형 -> ChartSpec 매핑
├── layout.md        # Composition Layer: 페이지 배치, 반응형, 상태 처리
└── theme.md         # Design Token Layer: 색/폰트/간격, 라이트/다크
```

### 4-2. 의존성 그래프

```text
main.md
  └─ data.md
       └─ indicators.md
            └─ insights.md
                 └─ charts.md
                      └─ layout.md ← theme.md
                           └─ Final Dashboard
```

### 4-3. 모듈별 책임 요약

| 모듈 | 책임 | 입력 | 출력 |
|------|------|------|------|
| main.md | 실행 흐름 정의 | 사용자 시나리오 | 모듈 호출 시퀀스 |
| data.md | 입력 정규화 | 카테고리/업로드 | MarketData |
| indicators.md | 지표 계산 | MarketData | 지표값 + 시그널 |
| insights.md | 자연어 해석 | 지표 + 시그널 | Insights[] |
| charts.md | 시각화 선택 | 데이터 유형 + 지표 | ChartSpec[] |
| layout.md | 화면 구성 | ChartSpec + Insights | LayoutSpec |
| theme.md | 디자인 토큰 | 정적/런타임 설정 | CSS 변수 / 차트 토큰 |

### 4-4. charts.md — 설명 가능한 자동 시각화

`charts.md`는 단순히 "어떤 차트를 쓸지"를 나열하지 않는다. 데이터 유형을 감지하고, 각 유형에 맞는 시각화 묶음을 자동 선택한다. 모든 ChartSpec은 `reason`과 `skillsRule`을 포함한다.

| 데이터 유형 | 자동 선택 차트 | 이유 |
|------------|----------------|------|
| OHLCV | candle + MA + volume + RSI + MACD | 가격/추세/모멘텀 동시 판단 |
| portfolio | donut + concentration bar | 비중과 쏠림 확인 |
| multi_asset | normalized comparison + correlation heatmap | 상대 성과와 분산 효과 확인 |
| returns | cumulative return + drawdown + monthly heatmap | 성과, 위험, 계절성 확인 |
| price_series | area/line + drawdown | 가격 흐름과 낙폭 확인 |

### 4-5. 인사이트 생성 — 외부 LLM API 미사용

`insights.md`는 IF/THEN 규칙과 텍스트 템플릿으로 구성된다.

```text
IF RSI < 20  -> "RSI {value}. 극단적 과매도, 반등 가능성 높음."
IF RSI 70~80 -> "RSI {value}. 과매수 구간. 단기 조정 가능성."
IF MA5 > MA20 > MA60 -> "이동평균 정배열로 상승 추세가 유지됩니다."
```

이 방식은 API 비용이 없고, 응답이 빠르며, Skills.md 자체가 분석 지능으로 동작한다는 점을 직접 보여준다.

| 바이브코딩 활용 | 15 | Skills.md -> 코드 매핑표, 프롬프트 로그, 재빌드 절차로 증거화 |
| 실용성/창의성 | 10 | 실시간 업데이트, 자동 시각화, 업로드 분석, 다크/라이트, 장애 fallback |

### 5-1. 바이브코딩 증거화 방식

개발 과정에서는 각 Skills.md 규칙이 어떤 코드 파일과 UI 기능으로 변환되었는지 매핑표로 기록한다. 또한 주요 프롬프트 로그와 재빌드 절차를 함께 정리하여, 결과물뿐 아니라 문서 기반 바이브 코딩 과정 자체를 재현 가능하게 증명한다.

## 6. 확장 기능 아이디어

Glancy의 확장 방향은 단순 기능 추가가 아니라, Skills.md 모듈을 교체하거나 추가하면 같은 데이터/지표/시각화 엔진 위에서 새로운 투자 분석 방식이 열리는 구조를 목표로 한다.

| 확장 기능 | 설명 | 연결되는 Skills.md |
|---|---|---|
| 투자 성향별 Skills 세트 | 보수적/공격적/단기/장기 투자 관점별 지표 기준, 가중치, 인사이트 문구를 교체한다. | `indicators.md`, `insights.md`, `theme.md` |
| 포트폴리오 리밸런싱 분석 | 업로드 포트폴리오의 비중 쏠림, 과거 성과, 리밸런싱 전후 위험/수익 변화를 비교한다. | `data.md`, `charts.md`, `insights.md` |
| 알림 규칙 엔진 | `insights.md`의 조건식을 가격, 지표, 변동성 알림 트리거로 확장한다. | `insights.md`, `indicators.md` |
| 다중 종목 비교 뷰 | 2~4개 종목의 기술 지표, 기본 지표, 상관관계, 상대 수익률을 한 화면에서 비교한다. | `data.md`, `charts.md`, `layout.md` |
| 다국어 인사이트 | `insights.md` 템플릿을 한국어/영어/일본어 등으로 전환해 글로벌 사용자에게 제공한다. | `insights.md` |
| 사용자 정의 Skills 업로드 | 사용자가 직접 작성한 Skills.md 번들을 업로드해 대시보드의 분석 기준과 화면 구성을 바꾼다. | `main.md`, `layout.md`, `theme.md` |

이 로드맵은 해커톤 제출 이후에도 현재 아키텍처를 버리지 않고 확장할 수 있음을 보여준다. 즉, Glancy의 핵심 자산은 특정 차트 몇 개가 아니라 “문서화된 분석 규칙을 실행 가능한 대시보드로 바꾸는 구조”다.

## 8. 기대 효과

Glancy는 투자 데이터를 단순히 보여주는 대시보드가 아니다. `Skills.md`가 데이터 처리, 지표 계산, 자연어 해석, 시각화 선택, 화면 구성, 디자인 토큰까지 지배하는 구조를 가진다. 따라서 본 프로젝트는 해커톤의 핵심인 Skills.md 설계, 대시보드 자동 생성, 바이브 코딩 활용, 실용적 시각화를 한 번에 증명한다.
